좋아요. **Korin 보안 전략 프롬프트 초안**은 이렇게 잡으면 됩니다.

OWASP Top 10은 웹 애플리케이션의 핵심 보안 위험 표준이며, ASVS는 개발·테스트 단계에서 확인할 수 있는 보안 요구사항 기준입니다. Korin은 이 두 기준을 기본 보안 프레임으로 잡는 것이 좋습니다. ([OWASP][1])

```text
Korin은 뉴질랜드 한인 커뮤니티 플랫폼이다. 
게시판, 회원가입/로그인, 중고거래, 구인구직, 댓글, 쪽지, 이미지 업로드, 사업자 광고, 회원등급 시스템을 포함한다.

이 웹사이트는 보안을 최우선으로 설계해야 한다. 
OWASP Top 10과 OWASP ASVS 기준을 기반으로, 다음 보안 원칙을 반드시 반영하라.

1. 인증 보안
- 이메일 인증 기반 회원가입
- 강력한 비밀번호 정책
- 비밀번호는 절대 평문 저장 금지
- bcrypt 또는 Argon2로 해싱
- 로그인 실패 횟수 제한
- 2단계 인증 옵션 제공
- 계정 탈취 의심 시 재인증 요구

2. 권한 관리
- 일반회원, 인증회원, 사업자회원, 관리자, 최고관리자 권한 분리
- 모든 API와 관리자 기능은 서버 측에서 권한 검증
- 사용자가 본인 게시글 외 타인의 글을 수정·삭제하지 못하도록 차단
- 관리자 페이지 URL 노출만으로 접근 불가하게 설계

3. 입력값 검증
- 모든 게시글, 댓글, 닉네임, 검색어, 이미지 설명, 프로필 입력값 검증
- SQL Injection 방지를 위해 ORM 또는 prepared statement 사용
- XSS 방지를 위해 HTML escaping과 sanitisation 적용
- 파일 업로드 시 확장자, MIME 타입, 파일 크기, 악성 스크립트 여부 검사

4. 세션 및 쿠키 보안
- 세션 ID는 예측 불가능하게 생성
- HttpOnly, Secure, SameSite 쿠키 적용
- 로그아웃 시 세션 무효화
- 장시간 미사용 시 자동 로그아웃
- JWT 사용 시 짧은 만료 시간과 refresh token 보호 적용

5. 게시판·커뮤니티 보안
- 스팸 글, 도배, 자동 가입 방지
- reCAPTCHA 또는 Turnstile 적용
- IP, 계정, 디바이스 기준 rate limiting 적용
- 신고 기능, 차단 기능, 관리자 검토 기능 제공
- 욕설, 사기, 피싱 링크 탐지 로직 적용

6. 중고거래·구인구직 안전
- 사기 신고 누적 계정 자동 제한
- 전화번호와 이메일 노출은 선택형으로 처리
- 개인정보는 기본적으로 비공개
- 거래 게시글에는 안전거래 안내 문구 포함
- 구인구직 게시판은 사업자 인증 또는 관리자 승인 옵션 제공

7. 개인정보 보호
- 최소한의 개인정보만 수집
- 비밀번호, 결제 정보, 민감정보는 저장하지 않거나 암호화
- 개인정보 처리방침과 이용약관 제공
- 사용자가 계정 삭제와 개인정보 삭제를 요청할 수 있게 설계
- 뉴질랜드 Privacy Act 원칙을 고려한 데이터 관리 구조 적용

8. 서버 및 인프라 보안
- HTTPS 필수
- Cloudflare 또는 WAF 적용
- DDoS 방어
- 관리자 페이지 IP 제한 또는 추가 인증
- 정기 백업
- 데이터베이스 접근 권한 최소화
- 환경변수로 API key와 secret 관리
- GitHub에 secret, DB 비밀번호, API key 업로드 금지

9. 관리자 보안
- 관리자 로그인 2FA 필수
- 관리자 활동 로그 기록
- 게시글 삭제, 회원 정지, 권한 변경 내역 기록
- 최고관리자 권한은 최소 인원만 보유
- 관리자 계정 탈취 시 전체 사이트가 위험하지 않도록 권한 분리

10. 모니터링 및 대응
- 비정상 로그인 탐지
- 대량 글쓰기, 대량 댓글, 반복 신고 패턴 감지
- 에러 로그와 보안 이벤트 로그 분리
- 의심 계정 자동 임시 제한
- 보안 사고 발생 시 관리자 알림
- 정기적인 보안 점검과 dependency 업데이트

개발 결과물은 단순히 기능이 작동하는 수준이 아니라, 보안 중심의 커뮤니티 플랫폼 구조로 설계해야 한다.
특히 인증, 권한, 입력값 검증, 파일 업로드, 스팸 방지, 개인정보 보호, 관리자 보안을 가장 우선순위로 둔다.
```

Korin은 특히 **회원 권한 관리, 게시글 스팸 방지, 중고거래 사기 방지, 관리자 계정 보호**가 핵심입니다. OWASP에서도 접근제어 실패, 암호화 실패, 인젝션, 인증 실패, 보안 설정 오류 등을 주요 위험으로 봅니다. ([OWASP][2])
