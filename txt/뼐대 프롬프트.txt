좋습니다. **AWS + Visual Studio Code + 추후 Android/iOS 앱 확장**을 기준으로 하면, Korin은 처음부터 **웹·앱 공통 백엔드 구조**로 설계하는 것이 좋습니다.

## 1. 추천 전체 구조

```text
Korin
= Next.js 웹사이트
+ AWS 백엔드
+ PostgreSQL 데이터베이스
+ S3 이미지 저장소
+ Admin 관리자 시스템
+ 추후 모바일 앱 공통 API 사용
```

AWS Amplify는 Next.js 앱 배포를 지원하고, RDS for PostgreSQL은 백업·복구·확장 같은 관리 기능을 제공하므로 1인 운영에 적합합니다. ([AWS Documentation][1])

# 3. 개발 방식 추천

혼자 개발하고 나중에 앱까지 만들려면 **순수 HTML/CSS/JS만으로는 한계**가 큽니다.

가장 추천하는 구조는:

```text
Frontend: Next.js
Backend API: Next.js API Routes 또는 별도 Node.js API
Database: PostgreSQL
Mobile App: React Native 또는 Flutter
```

이렇게 하면 웹과 앱이 같은 API를 사용할 수 있습니다.

```text
웹사이트  →  Korin API  →  Database
앱        →  Korin API  →  Database
관리자    →  Korin API  →  Database
```

# 14. 개발 프롬프트 최종본

```text
Korin은 뉴질랜드 한인 커뮤니티 플랫폼이다. 
AWS 환경에서 호스팅되며, Visual Studio Code를 IDE로 사용한다. 
초기에는 웹사이트로 개발하지만, 추후 Android 및 iOS 앱으로 확장될 수 있도록 API 중심 구조로 설계한다.

기술 구조는 Next.js, TypeScript, PostgreSQL, AWS Amplify Hosting, Amazon RDS, Amazon S3, Amazon Cognito, CloudFront, AWS WAF를 기준으로 한다.

프로젝트는 유지보수가 쉽도록 공통 컴포넌트, 공통 CSS, 카테고리별 페이지, API, 타입 정의, 설정 파일, 데이터베이스 스키마를 명확하게 분리한다.

공통적으로 사용하는 Header, Footer, Button, Card, Modal, SearchBar, PostCard, PostList, PostForm 등은 components/common 또는 components/post 폴더에 관리한다.

CSS는 globals.css, variables.css, layout.css, buttons.css, forms.css, board.css, admin.css처럼 공통 스타일과 기능별 스타일을 분리한다.

각 게시판은 완전히 별도의 DB를 갖는 방식이 아니라, posts 테이블과 categories 테이블을 중심으로 category_id를 통해 구분한다. 이를 통해 자유게시판, 중고거래, 구인구직, 렌트, 자동차, 업체 홍보 게시판을 같은 구조로 관리할 수 있게 한다.

회원 권한은 guest, member, verified_member, business_member, moderator, admin, super_admin으로 나눈다.

일반회원은 글쓰기, 댓글, 찜, 신고 등의 기능을 사용할 수 있다. 이메일 인증회원은 중고거래와 쪽지 기능을 사용할 수 있다. 사업자회원은 구인공고와 업체 홍보를 등록할 수 있다.

관리자 페이지는 /admin 경로에 별도로 구성한다. 관리자는 회원 관리, 게시글 관리, 댓글 관리, 신고 관리, 광고 설정, 카테고리 관리, 사업자 인증, 공지사항, 보안 로그, 사이트 설정을 관리할 수 있어야 한다.

관리자 계정은 일반회원과 구분되며, 2단계 인증, 활동 로그, 권한 분리, 로그인 실패 제한을 적용한다. 모든 관리자 행동은 admin_logs 테이블에 기록한다.

광고 시스템은 ads 테이블을 사용하며, 광고 제목, 이미지, 링크, 위치, 시작일, 종료일, 활성 상태, 클릭 수를 관리한다. 관리자는 광고 위치를 메인 상단, 게시글 사이, 사이드바, 모바일 하단 등으로 설정할 수 있다.

이미지 업로드는 Amazon S3를 사용하며, 게시글 이미지, 프로필 이미지, 광고 이미지를 분리해서 관리한다.

모든 기능은 추후 모바일 앱에서 동일하게 사용할 수 있도록 REST API 또는 GraphQL API 중심으로 설계한다.

보안은 OWASP 기준을 따르며, 입력값 검증, XSS 방지, SQL Injection 방지, 파일 업로드 검증, rate limiting, WAF, CloudFront, 관리자 로그, 신고 시스템을 포함한다.

최종 목표는 혼자서도 유지보수 가능한 구조, 빠른 배포, 쉬운 확장, 강한 보안, 앱 확장성을 모두 갖춘 커뮤니티 플랫폼을 만드는 것이다.
```
