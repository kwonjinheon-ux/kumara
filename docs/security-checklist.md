# Colink.nz Security Checklist

Use this checklist before merging high-risk features and before production
launch.

## Authentication

- [ ] Email verification is required for sensitive features.
- [ ] Email sign up completes email-code verification before account creation.
- [ ] Sign up is protected by reCAPTCHA in configured environments.
- [ ] Admin MFA is required.
- [ ] Login attempts are rate-limited.
- [ ] Suspicious repeated failures trigger lock, delay, or challenge behavior.
- [ ] Sensitive actions require re-authentication.

## Authorization

- [ ] Every API route checks the current user role.
- [ ] Mutations check ownership where relevant.
- [ ] Admin routes check explicit admin permissions.
- [ ] Role changes are logged.
- [ ] UI visibility is not treated as access control.

## Inputs

- [ ] API bodies are schema-validated.
- [ ] Query strings and path params are validated.
- [ ] Unexpected fields are rejected or ignored deliberately.
- [ ] SQL queries use prepared statements, an ORM, or a query builder.
- [ ] User-generated content is escaped or sanitized.

## Uploads

- [ ] Uploads use short-lived S3 pre-signed URLs.
- [ ] Object keys are generated by the server.
- [ ] MIME type, extension, size, and dimensions are validated.
- [ ] Executable file types are blocked.
- [ ] Profile, post, and ad uploads use separate prefixes.

## Community Safety

- [ ] Posting and commenting are rate-limited.
- [ ] Reports can be submitted and reviewed.
- [ ] Moderators can hide posts and comments.
- [ ] Repeated fraud or abuse reports can restrict an account.
- [ ] Contact details are private by default.

## Admin

- [ ] Admin sessions have stricter timeout behavior.
- [ ] High-risk admin actions are logged.
- [ ] `super_admin` is limited to the minimum number of accounts.
- [ ] Admin login failures are monitored.
- [ ] Admin logs include actor, action, target, IP address, and timestamp.

## Infrastructure

- [ ] HTTPS is enforced.
- [ ] RDS is not publicly exposed.
- [ ] Secrets are not committed.
- [ ] AWS IAM permissions are least-privilege.
- [ ] CloudFront, WAF, CloudWatch, and backups are configured for production.
