# Colink.nz Recent Implementation Updates

이 문서는 최근 화면과 기능 변경사항 중 설계 문서에 흩어지기 쉬운 내용을 한곳에
정리한다. 이후 세부 설계는 각 문서로 옮기되, 현재 구현 기준을 빠르게 확인할 때
이 문서를 참고한다.

## Marketplace

Route:

```text
/marketplace
/marketplace/new
/marketplace/:postId
```

현재 마켓플레이스는 Reddit형 데스크톱 구조와 모바일 최적화 구조를 함께 사용한다.
데스크톱은 좌측 마켓 메뉴, 중앙 피드, 오른쪽 가격/상태 정보를 중심으로 구성한다.
모바일은 플로팅 햄버거 메뉴, 상단 검색/판매등록, 카드형 피드로 구성한다.

최신 UI 기준:

```text
- 모바일 가로 스크롤 카테고리 탭 제거
- 플로팅 햄버거 메뉴로 마켓 메뉴 열기
- 검색창과 판매등록 버튼 높이 통일
- 판매등록 버튼은 공통 CTA 스타일과 사선 shine hover 사용
- 제목 옆 댓글 수는 작은 원형 숫자 배지로만 표시
- 조회수는 작성자/시간 라인 옆에 한 번만 표시
- 물품상태, 거래방식, 지역은 본문 정보 라인에 표시
- 가격은 카드 오른쪽 또는 모바일 오른쪽 하단에 강하게 표시
- 판매 상태는 모바일에서 썸네일 아래에 표시
- 끌어올림 표시는 썸네일 위 작은 불꽃 아이콘만 사용
- 목록 카드에서는 수정, 삭제, 끌어올리기 버튼 미노출
```

## Marketplace Detail

상세 페이지는 읽기 흐름을 우선한다.

```text
- 제목, 판매상태, 작성자, 작성일, 조회수를 상단에 배치
- 작성자 닉네임 앞에 둥근 프로필 이미지 표시
- 가격, 지역, 거래방식, 물품상태는 미니멀 정보 영역으로 표시
- 파란 테두리형 정보 그리드는 사용하지 않음
- 내부 이미지 파일명/식별자 미노출
- 데스크톱 갤러리는 큰 이미지 + 고정 위치 화살표 + 하단 썸네일 사용
- 모바일은 이미지를 본문 흐름에 세로로 자연스럽게 나열
- 북마크 클릭 시 즉시 상태 변경 및 하트 burst 애니메이션
- 참여 버튼은 기본 상태에서 가볍게, hover 시 강조
```

## Marketplace Writing

글쓰기는 단순 textarea가 아니라 블로그형 HTML 편집 흐름으로 간다.

```text
- HTML 편집 도구는 버튼을 눌러 펼침
- 모바일에서는 처음부터 긴 도구 모음을 노출하지 않음
- H2/H3, 굵게, 기울임, 밑줄, 취소선, 인용, 목록, 번호 목록 지원
- 좌측/가운데 정렬, 링크/해제, 구분선, 서식삭제 지원
- 기본 색상 10가지 제공
- 본문 중간 이미지 삽입 지원
- 본문 이미지 크기 프리셋 조절 지원
- 본문 이미지 파일 선택 후 마지막 커서 위치에 삽입
- 본문 이미지 마우스 리사이즈 지원
- 편집기 안내 문구는 포커스 시 사라짐
```

첨부 이미지:

```text
- 본문 아래 여러 장 첨부 가능
- 대표 이미지 지정 가능
- 대표 이미지는 이미지 위 `대표` 배지로 표시
- 각 이미지는 오른쪽 위 X 버튼으로 개별 삭제
- 이미지 선택 버튼은 회색 계열 미니멀 버튼과 hover 애니메이션 사용
- 대표 이미지는 목록 썸네일로 사용
- 첨부 이미지 프리뷰 마우스 크기 조정 지원
- 업로드 전 1280px 본문 이미지와 420px 썸네일로 리사이즈
```

등록 중 UX:

```text
- 글 등록 중 화면 중앙 modal progress overlay 표시
- 문구: 글을 등록중입니다
- 업로드 진행률은 XMLHttpRequest upload progress 기준으로 표시
- 성공 시 상세 페이지 이동 전까지 overlay 유지
```

연락 방식:

```text
- Figma 스타일 checked-option panel
- 흰색 옵션, 얇은 border, 8px radius
- 민트 선택 상태와 작은 hover elevation
- 카드형 과장보다 단순하고 명확한 체크 옵션을 우선
```

## My Page

Route:

```text
/my-page
```

내 페이지는 프로필, 게시글, 댓글, 북마크, 채팅, 키워드 알람, 보안, 회원등급을
관리하는 계정 허브다.

현재 유지할 기준:

```text
- 본문 영역은 과한 bold를 피하고 normal 또는 중간 굵기 사용
- 왼쪽 메뉴도 같은 font weight 기준 적용
- 닉네임과 아이디(이메일)는 분리 표시
- 이메일은 수정 불가 항목
- 프로필 이미지는 원형 표시 및 업로드/조정 가능
- 지역은 드롭다운, 세부 지역은 선택 지역 기반 드롭다운
```

## Interaction Performance

반응 속도는 핵심 UX 기준이다.

```text
- 북마크, 추천, 상태 변경, 댓글 등록은 optimistic UI로 즉시 반응
- 목록 조회는 가벼운 요약 데이터 사용
- 상세 페이지에서만 전체 본문과 이미지 데이터 사용
- 모바일에서 무거운 아코디언 렌더링 지양
- 스크롤 영역을 분리해 전체 페이지 레이아웃 재계산 최소화
- 글쓰기 이미지는 클라이언트 리사이즈 후 전송해 JSON payload를 줄임
- post store는 메모리 캐시와 file mtime으로 stale cache를 방지
- 목록/상세 HTML에는 data:image base64 payload를 직접 싣지 않고 이미지 API URL만 전달
- 마켓 이미지 API는 public immutable cache header로 반복 조회 비용을 낮춤
- 상세 조회에서 ID 미발견 시 disk force refresh 후 404 처리
- 다음 단계는 presigned direct upload + DB insert + async job queue
```

## 2026-05-10 Marketplace Detail And Mobile Consistency

최근 상세 페이지와 모바일 마켓플레이스 기준:

```text
- 상세 페이지 본인 글 관리 버튼은 지역 정보 박스와 같은 흰 배경, 연한 파란 테두리, 진한 텍스트 톤 사용
- 상세 제목, 본문, 댓글 폰트 크기는 기존 기준으로 유지
- 목록과 상세 페이지는 이미지 data URL을 HTML에 직접 직렬화하지 않고 /api/marketplace/posts/:postId/images/:imageId 경유
- 목록 HTML은 약 38.6MB에서 약 108KB 수준으로 축소
- 상세 HTML은 약 91KB 수준이며 data:image 직렬화 0개 유지
- 모바일 마켓 카드는 420px 이하에서도 iPhone Pro Max와 같은 3열 구조 유지
- Android Galaxy, iPhone, in-app browser 모두 같은 모바일 레이아웃 규칙을 사용
- 브라우저별 자동 글자 확대 차이를 줄이기 위해 text-size-adjust를 100%로 고정
```

## Common Button Rule

Korin의 주요 버튼은 공통 CTA 스타일을 따른다.

```text
- 밝은 파란색 계열
- pill 형태
- 과한 그림자 없음
- hover 시 한두 줄의 사선 shine 애니메이션
- press 시 살짝 눌리는 느낌
- focus-visible 상태 명확히 표시
```

버튼 애니메이션은 사용자가 눌렀을 때 기분 좋은 피드백을 주되, 반복적인 줄무늬나
무거운 그림자로 화면을 산만하게 만들지 않는다.

## 2026-05-02 Layout And Threading Updates

마켓 댓글 트리는 모바일과 데스크탑에서 같은 좌표 체계를 사용한다.

```text
- 부모 댓글의 세로선은 아바타 중앙 x축에서 시작한다.
- 대댓글 연결선은 부모 세로선에서 출발해 자식 아바타 테두리까지만 이어진다.
- 연결선은 아바타 원형 내부로 들어가지 않는다.
- 댓글 깊이별 아바타 크기는 root > reply > nested reply 순서로 작아진다.
- 추가 들여쓰기는 CSS margin이 아니라 댓글 body 내부 중첩 구조로 만든다.
```

마이페이지 메뉴는 URL 쿼리와 내부 상태를 동시에 갱신한다.

```text
- /my-page?tab=profile
- /my-page?tab=posts
- /my-page?tab=comments
- /my-page?tab=bookmarks
- /my-page?tab=chat&chatId={id}
- /my-page?tab=keywords
- /my-page?tab=security
- /my-page?tab=membership
```

프로필 드롭다운, 모바일 플로팅 메뉴, 데스크탑 사이드바에서 같은 탭 매핑을 사용해
로그인 직후에도 메뉴가 흔들리거나 다른 탭으로 엇갈리지 않게 유지한다.

## 2026-05-03 Comment Thread Finish

댓글 스레드에서 펼쳐진 답글 목록은 자식 댓글을 먼저 렌더링하고, `답글 숨기기` 토글은
스레드의 마지막에 배치한다. 이 순서를 기준으로 세로 연결선이 답글 묶음 끝까지 자연스럽게
내려간 뒤 마무리되어, 모바일과 데스크탑 모두에서 댓글 선이 버튼 중간에 끊겨 보이지 않도록 한다.

답글 입력 폼은 카드형 박스 대신 선형 입력 UI를 사용한다. 취소 버튼은 노출하지 않고 `답글 등록`
버튼만 남겨, 댓글 흐름 안에서 가볍게 답글을 작성하는 느낌을 유지한다.

## 2026-05-03 Comment Controls And My Page Sync

댓글 헤더의 `•••` 메뉴는 작성자에게만 보이며, 같은 댓글 API에서 수정과 삭제를 처리한다.
내 페이지의 탭 상태는 `tab`과 `chatId` 쿼리스트링을 다시 읽어 동기화하므로 로그인 후
프로필 메뉴나 사이드바에서 이동해도 데스크탑과 모바일이 같은 카테고리를 보여준다.

댓글 연결선은 펼쳐진 스레드에만 표시하고, 접힌 스레드는 왼쪽 세로선을 완전히 숨긴다. 답글
입력창은 커서가 선 바로 위에서 시작되도록 패딩을 줄였고, 하단 스트로크는 아주 연하게 유지한다.

## 2026-05-03 Comment Thread Refinement

댓글과 대댓글 연결선은 모바일 override에서도 데스크탑과 같은 좌표 계산을 사용한다.

```text
- 펼쳐진 스레드의 세로선은 답글 숨기기 토글까지 부드럽게 내려간다.
- 접힌 스레드는 세로선과 자식 영역을 모두 숨겨 왼쪽 잔상이 남지 않는다.
- 대댓글 곡선은 부모 라인에서 출발해 자식 아바타 테두리까지만 이어진다.
- 답글 입력창은 카드가 아니라 얇은 하단 스트로크를 가진 미니멀 입력 UI로 유지한다.
- 댓글 작성자 관리 메뉴는 헤더 오른쪽의 점 3개 버튼에서 수정/삭제를 제공한다.
```

## 2026-05-03 Comment Thread Micro Layout

댓글 스레드의 모바일 override도 부모 아바타 중심 좌표를 기준으로 연결선을 계산한다.
대댓글로 들어가는 곡선은 부모 세로선에 바로 붙고, 접힌 스레드는 세로선과 곡선을 모두 숨긴다.

답글 입력은 선형 UI로 유지한다. 커서가 하단 스트로크 가까이에서 시작되도록 입력 높이와 패딩을
줄이고, 스트로크는 거의 보이지 않는 수준으로 낮춰 댓글 흐름 안에서 가볍게 보이도록 한다.
